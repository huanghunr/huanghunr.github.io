三七彩票

2024/12/19 算法 算法
  • 问题分析
  • 思路
  • 算法步骤
  • 伪代码
  • 解释
  • 时间复杂度
  • 测试用例

这个问题是一个涉及图论的问题，具体来说，可以通过图的连通性和拓扑排序来解决。我们可以将每一天的参与者和中奖者关系抽象成一个图，分析每天的抽奖问题。

问题分析

  1. 参赛者与中奖者：在每一天 i，有一组参与者。如果某个人在某一天中奖，那么这个人不能在之后的任何一天再次中奖。因此，我们可以将这些参与者看作是一个集合，每次抽奖的结果会减少当前集合的大小，并影响后续抽奖的集合。
  2. 问题的本质：实际上这是一个约束满足问题。每一天的抽奖要求从当天的参与者中选择一个中奖者，且不能重复选择之前的中奖者。问题可以通过建立一个合适的排除规则来解决，类似于图的拓扑排序。

思路

  1. 图的构建：
    • 每一天的中奖者可以看作是一个节点，所有的参与者都和这个节点有一条“边”，表示他们可以被选为中奖者。
    • 题目给出的要求类似于一个“排除图”，即如果某个人在某一天中奖，就不能出现在之后的任何一天。
  2. 使用贪心算法：
    • 每一天选择一个合适的中奖者，使得在后续天数中不会违反规则。我们可以使用一个优先队列（最小堆）来选择最小的参与者编号，从而避免后续冲突。
  3. 贪心选择的核心：在每一天的候选中奖者中，选择一个最小的编号，然后将这个中奖者从后续天数的候选名单中删除。
  4. 具体步骤：
    • 使用一个数组或哈希集来维护每个人的“参与情况”，即记录哪些人已经被选中过。
    • 遍历每一天的参与者，挑选一个最小的、没有被选中过的参与者作为中奖者。
  5. 判断是否有解：
    • 如果在某一天没有可选的中奖者，那么说明没有解决方案，输出 -1。
    • 否则，输出一个可能的中奖者列表。

算法步骤

  1. 解析每个测试用例。
  2. 对每一天的参与者构建集合。
  3. 使用优先队列（最小堆）来选择合适的中奖者。
  4. 如果某天无法选出中奖者，则输出 -1。
  5. 否则，输出各天的中奖者编号。

伪代码

#include <stdio.h>
#include <stdlib.h>
#include <stdbool.h>

#define MAXN 50000

// 优先队列比较函数，用于最小堆
int cmp(const void* a, const void* b) {
    return (*(int*)a) - (*(int*)b);
}

int main() {
    int t;
    scanf("%d", &t);

    while (t--) {
        int m;
        scanf("%d", &m);

        int winners[m];  // 存储每一天的中奖者
        bool chosen[MAXN] = {false};  // 标记某人是否已中奖

        bool possible = true;
        for (int i = 0; i < m; i++) {
            int ni;
            scanf("%d", &ni);

            int participants[ni];
            for (int j = 0; j < ni; j++) {
                scanf("%d", &participants[j]);
            }

            // 排序参与者，选择未中奖的最小值
            qsort(participants, ni, sizeof(int), cmp);
            bool found = false;
            for (int j = 0; j < ni; j++) {
                if (!chosen[participants[j]]) {
                    winners[i] = participants[j];
                    chosen[participants[j]] = true;
                    found = true;
                    break;
                }
            }

            if (!found) {
                possible = false;
                break;
            }
        }

        if (possible) {
            for (int i = 0; i < m; i++) {
                printf("%d ", winners[i]);
            }
            printf("\n");
        } else {
            printf("-1\n");
        }
    }

    return 0;
}

解释

  1. 输入解析：
    • 首先读取测试用例的数量 t。
    • 对于每个测试用例，首先读取天数 m，然后对于每一天，读取参与者数量 ni，再读取该天的参与者编号。
  2. 排序与选择：
    • 对当天的参与者进行排序（使用 qsort）。然后从排序后的数组中选择第一个未被选择的参与者作为中奖者，确保没有重复中奖者。
  3. 标记中奖者：
    • 使用 chosen 数组来标记每个人是否已经中奖。如果某个参与者已经中奖，则跳过这个人，选择下一个未中奖的人。
  4. 判断是否有解：
    • 如果某一天无法选出未中奖的参与者，则输出 -1，表示无解。
    • 如果可以为每一天选出中奖者，则输出这些中奖者的编号。

时间复杂度

  • 每个测试用例中对于每一天的参与者进行排序，排序的复杂度是 O(ni log ni)，其中 ni 是第 i 天的参与者数量。总的时间复杂度是 O(sum(ni) log ni)，其中 sum(ni) 是所有天数参与者数量的总和。由于总和不超过 50000，排序的复杂度是可接受的。

测试用例

  1. 输入：

    1
3
3
1 2 3
2
2 3
3
3 1 2

    输出：

    1 2 3

  2. 输入：

    1
3
3
1 2 3
1
3
2
1 2

    输出：

    -1

这个方法通过贪心策略和排序来保证在每一天都能够选择一个合适的中奖者，同时确保解的正确性。
